画像生成AIを活用するための日本語一言プロンプト集を作成するプロンプトを考案したいです。前提として、画像が1枚以上入力されていること。パラメータとして、画像の枚数、内容、どういうプロンプトが欲しいか方針を設定します。これにより、パラメータに該当する日本語プロンプト集を作成できるように、プロンプトを作成したいです。


# フェーズ1: 初期分析と改善提案

## ステップA：プロンプト診断

### 1. **目的の明確性** ✓ 良好
- 画像生成AI用の日本語一言プロンプトをK個生成することは明確
- 最終用途（画像生成AIへの直接入力）も明示されている

### 2. **不足要素** ⚠️ 重大
- **インプットパラメータが空欄**：実際に入力される値の例や期待形式が不明
- **「深く参照」の具体的方法**：参照画像をどう解釈するかの指示がない
- **品質基準**：生成されたプロンプトの良し悪しを判断する基準が未定義
- **「任意の画像生成AI」の曖昧性**：DALL-E、Midjourney、Stable Diffusionで最適なプロンプトは異なる
- **「一言」の定義**：文字数上限や構造的制約が不明確

### 3. **曖昧表現** ⚠️ 改善必要
- **「深く参照」**：どの程度詳細に？どの要素を優先？
- **「簡潔な一言」**：20文字？50文字？複数の名詞句の連結は許容？
- **「できる限り多様な」**：どのような軸で多様性を担保？（構図、色彩、雰囲気など）
- **「方針の範囲内で」**：方針が空欄の場合の処理は？

### 4. **実装上の課題** ❌ 高リスク
- **空欄パラメータの処理**：AIが空欄を見た場合、どう対応すべきか不明
- **出力検証の困難性**：K個が正確に出力されたか機械的に確認しにくい形式
- **多様性の保証**：「多様性」の定義が曖昧なため、似たプロンプトが生成される可能性

## ステップC：改善版プロンプト（提案1〜3を統合）

````markdown
# 厳格なプロンプトジェネレーター：日本語一言プロンプト集 作成用

## 1. あなたの役割と前提
あなたは、**画像生成AI**の特性を熟知した**日本語プロンプト設計の専門家**です。

## 2. インプットパラメータ
以下の情報に基づき、タスクを実行してください。

### 【インプットデータ】
- **参照画像枚数（N）**：  [例：3枚／0の場合はテキスト情報のみで生成]
- **参照画像の内容**：  [例：「夕暮れの海岸、シルエット人物、暖色系」／画像がない場合は「なし」]
- **プロンプトの設計方針**：  [例：「ノスタルジックで感傷的な雰囲気」／未指定の場合は「参照画像の特徴を忠実に反映」]
- **生成プロンプトの数（K）**：  [例：10個／必須入力]

## 3. タスクと要件
上記のインプットを**深く参照**し、画像生成AIに直接入力可能な高品質な**日本語一言プロンプト**を**正確にK個**考案してください。

### 【厳守事項】
- **言語**：日本語のみ
- **形式**：30文字以内、句点なし、名詞句の並列は3要素まで
- **出力数**：**Kの数に厳密に従うこと**（過不足は許容されない）
- **多様性の保証**：K個のプロンプトは以下の軸で変化をつけること
  - 構図（俯瞰/クローズアップ/全体など）：最低3パターン
  - 時間帯・照明（昼/夜/逆光など）：最低2パターン
  - 表現スタイル（写実/イラスト調/抽象など）：最低2パターン
  ※K<7の場合は可能な範囲で多様性を確保

## 4. 出力形式
以下の形式で出力してください。説明文は一切不要です。

```
プロンプト1
```

```
プロンプト2
```

...

```
プロンプトK
```

**生成数確認：K個**
````
