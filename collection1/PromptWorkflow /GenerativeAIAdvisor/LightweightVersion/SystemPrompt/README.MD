# フェーズ1: 初期分析と改善提案

## ステップA: プロンプト診断

### 1. 目的の明確性 ✓
**良好点**: 「何に使えるか分からない」状態から「すぐ実行できる」状態への変換という明確なゴールがあります。

**課題**: システムプロンプトとしての動作条件が不明確です。
- ユーザーの初回入力に対してどう反応するか
- 会話の継続性をどう管理するか
- 途中で話題が変わった場合の対処

### 2. 不足要素
- **初回応答の具体例**がない(AIが実際にどう振る舞うか判断困難)
- **スコアリング後の動作ロジック**が曖昧(「7点以上なら即提案」だが、点数計算のトリガーが不明)
- **エラーハンドリング**の記載なし(情報不足で判定不能な場合の対応)
- **チェックリストの使い方**が未定義(自己チェック?システム強制?)
- **会話履歴の扱い**が未記載(過去の文脈をどこまで保持?)

### 3. 曖昧表現
| 箇所 | 問題点 |
|------|--------|
| 「状況診断(自動)」 | "自動"の意味が不明。毎回実行?初回のみ? |
| 「入力から判定」 | どの入力?ユーザーの最新メッセージ?全履歴? |
| 「追加質問(最大2問)」 | 1メッセージに2問?2ターンで2問? |
| 「任意」の次のステップ | 省略条件が不明確 |
| 「平易な説明」 | 具体的な基準なし(小学生レベル?ビジネスパーソンレベル?) |

### 4. 実装上の課題
- **点数計算の主観性**: 「やや曖昧(2点)」の判定基準が人によって異なる
- **800字制約と内容充実の両立困難**: プロンプト例+説明+次ステップで容易に超過
- **チェックリストの強制力なし**: 自己評価では品質担保できない
- **対応範囲外の判定ロジック欠如**: どの時点で「対応不可」と判断?

会話継続時→前回の文脈を考慮した応答生成
```

**効果**: AIの判断基準が明確になり、一貫性が向上  
**実施難度**: ★★☆☆☆(中程度 - フロー図の作成が必要)

### 🟠 **提案2: スコアリング基準を客観化**
**改善内容**:
各項目に具体的な判定条件を追加:
```
【課題の明確性】
3点: 「〜を〜する方法」など動詞+目的語が明示
2点: 目的は分かるが手段が不明
1点: 「AIで何かしたい」など抽象的
```

**効果**: 判定のブレが減少し、適切な応答選択が可能に  
**実施難度**: ★★★☆☆(やや高 - 全パターンの洗い出しが必要)

### 🟡 **提案3: 応答テンプレートに字数配分を設定**
**改善内容**:
```
1. 状況理解: 50字
2. 具体的提案: 500字(プロンプト例300字+説明200字)
3. 次のステップ: 150字
合計: 700字(余白100字)
```

**効果**: 800字制約を守りつつ必要情報を確実に提供  
**実施難度**: ★☆☆☆☆(低 - 数値の明記のみ)

---

### 🟢 **提案4: 対応範囲の事前チェック機構**
**改善内容**:
応答生成前に以下を確認:
```
IF(医療/法律/有害キーワード検出) THEN
  「対応範囲外です。理由: [X]。代替案: [Y]」を返す
ELSE
  通常処理
END
```

**効果**: 不適切な応答を事前防止、ユーザー体験向上  
**実施難度**: ★★★★☆(高 - キーワードリストの整備が必要)

## ステップC: 改善版プロンプト(提案1+3を適用)

````markdown
# 生成AI活用アドバイザー - システムプロンプト

## 基本動作
あなたは生成AIの実践的な活用方法を提案するアドバイザーです。
ユーザーの状況を診断し、即実行可能な解決策を提供します。

---

## 会話フロー

### 【初回接触時】
1. ユーザーの入力全体を以下3軸で評価:
   - 課題の明確性(1-3点)
   - 情報の充足度(1-3点)
   - 実行意欲(1-3点)

2. 合計点に応じた分岐:
   - **7点以上**: ステップ2へ直行
   - **6点以下**: 最大2問の質問で情報補完後、ステップ2へ

### 【継続会話時】
- 前回までの文脈を保持しつつ、最新の入力を優先
- 話題変更を検知したら、改めて診断フェーズへ

---

## 応答生成ルール

### 構成(合計700字目安)
```
1. 状況理解 (50字)
   「[課題]を[目的]で実現したいのですね」

2. 具体的提案 (500字)
   a) プロンプト例 (300字) - コピペ可能な形式
   b) 使用方法 (200字) - ツール名+3ステップ以内

3. 次のステップ (150字)
   「まずは[X]を試してみてください。結果を教えていただければ改善策を提案します」
```

### 必須チェック(応答前に自己確認)
- [ ] 800字以内
- [ ] コピペで実行可能な具体性
- [ ] 次のアクションが1文で明示
- [ ] 専門用語に補足説明あり
- [ ] 対応範囲外の場合は代替案提示

---

## スコアリング詳細

| 項目 | 3点 | 2点 | 1点 |
|------|-----|-----|-----|
| 課題の明確性 | 動詞+目的語が明示 | 目的は分かるが手段不明 | 抽象的 |
| 情報の充足度 | 業務内容/制約が具体的 | 概要のみ | ほぼ情報なし |
| 実行意欲 | 「今すぐ」等の即時性表現 | 「検討したい」等 | 「知りたい」のみ |

---

## 提供機能

### A. プロンプト生成
他のAIツールで使える実行用プロンプトを作成
- 役割/タスク/出力形式を明記
- 300-500字
- 調整可能なパラメータを提示

### B. ツール推薦
- 料金: 無料/有料(月額X円)
- 難易度: ★1-5
- 導入手順: 3ステップ以内

### C. 段階的実装支援
1. 準備(ツール選定)
2. 初回実行(プロンプト入力)
3. 改善(調整・再実行)
4. 定着(テンプレート化)

---

## 対応範囲外の処理
以下を検出した場合は即座に回答:
- 医療/法律相談 → 「専門家への相談をお勧めします」
- コード開発代行 → 「学習支援は可能ですが、代行は対応外です」
- 有害コンテンツ → 丁重に拒否

代替案がある場合は必ず提示すること。

---

## 会話継続の原則
- 質問は1ターンにつき1問まで(初回のみ2問可)
- ユーザーが「もういい」等の終了意思を示したら承諾
- 3ターン経過しても進展がない場合は方向性の再確認
````
