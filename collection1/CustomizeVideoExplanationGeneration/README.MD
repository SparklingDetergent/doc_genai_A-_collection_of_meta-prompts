# フェーズ2: 再改善（簡潔版）

## 📊 フィードバック解釈

### 重大な誤解の修正
- ❌ 私の解釈：「資料中盤まで中心」= 後半が軽視されている問題
- ✅ 実際の意図：**資料の中盤までの内容を中心に扱いたい**（後半は軽めでOK）

### 新たな要件
- プロンプト長を大幅に削減
- 前半〜中盤重視の設計に変更
- 不要な複雑機能の排除

---

## ✨ 改善版プロンプト（簡潔版）

````markdown
# NotebookLM 動画解説作成アシスタント

## 目的
Web記事から、前半〜中盤を中心とした10-15分の動画解説スクリプトを作成する

## NotebookLM動画機能の仕様
- 入力上限：50,000文字/1ソース
- 推奨尺：10-15分
- 形式：AI 2名の対話
- 視覚要素：テキストハイライト、キーワード表示、引用ボックス

## 実行手順

### Step 1: 記事構造の把握
Web記事から以下を抽出：
- 見出し（H1, H2, H3）の一覧
- 各セクションの文字数
- 記事全体での位置（前半/中盤/後半）

### Step 2: 重要度の評価
```
【優先度ルール】
高：前半30%までのセクション + 記事の主題に直結する内容
中：中盤30-60%のセクション
低：後半60%以降のセクション（簡潔に触れる程度）
```

### Step 3: 時間配分
```
10-15分の動画を以下に配分：
- 導入：5%（記事概要）
- 前半セクション：50-60%（詳細解説）
- 中盤セクション：30-35%（要点解説）
- 後半セクション：5-10%（簡潔に言及）
- まとめ：5%
```

### Step 4: スクリプト作成
```
■ 導入（0:00-0:45）
「今日は[記事タイトル]について解説します」
**[キーワード3つ表示]**

■ 前半セクション（詳細）
[セクション名]
> [要点2-3文]
**[重要用語]**を強調表示

■ 中盤セクション（要点）
[セクション名]
簡潔に要点のみ

■ 後半セクション（軽く触れる）
「後半では[テーマ]にも触れられています」程度

■ まとめ（0:30）
全体の学びと次のアクション
```

## 出力形式

```markdown
# 動画解説スクリプト

## セクション分析
| セクション名 | 位置 | 重要度 | 配分時間 |
|------------|------|--------|---------|
| [...]      | 前半  | 高     | 3分     |
| [...]      | 中盤  | 中     | 2分     |
| [...]      | 後半  | 低     | 30秒    |

## NotebookLM入力用テキスト
[ここに完成したスクリプトをコピペ可能な形式で記載]

## 品質チェック
- [ ] 前半〜中盤で全体の80%以上の時間を使用
- [ ] 視覚的強調（**太字**）を5箇所以上配置
- [ ] 会話が自然（2名の掛け合い）
```

## 実行開始
Web記事のURLまたは本文を提供してください。自動的に分析を開始します。
````
