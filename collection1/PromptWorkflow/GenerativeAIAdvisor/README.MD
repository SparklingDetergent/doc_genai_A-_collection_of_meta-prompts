# 生成AIアドバイザー プロンプト要件定義書

## 目的
生成AIの活用シーンが分からないユーザーに対し、生成AI自身が使い方を提案・支援する仕組みを構築する。

## 前提条件
- AIチャットによるテキストベースの対話環境を想定
- ユーザーの入力内容に応じて、生成AIで解決策を提示

## 動作モード

### 1. 処理方式による分類

**自己完結型（単独処理モード）**
- AIが全ての情報を集約し、単独で問題解決を完結させる
- ユーザーに追加情報の提供を求める

**司令塔型（マルチエージェントモード）**
- AI自身は直接処理せず、アドバイザーとして振る舞う
- 他の生成AIツールや情報源の活用方法を指示

### 2. 対話スタイルによる分類

**テーマ型**
- 特定の課題やテーマを中心に、集中的に対応
- 明確な目標がある場合に適用

**伴走型**
- 継続的な対話を通じて、段階的にアドバイスを提供
- プロセス全体をサポート

## 実装方針
上記4パターン（自己完結型×テーマ型、自己完結型×伴走型、司令塔型×テーマ型、司令塔型×伴走型）を統合した単一ワークフローとして設計し、ユーザーの状況に応じて自動的に最適なモードで対応する。
