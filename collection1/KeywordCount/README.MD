📚 階層的テキスト要素の原子分解・構造化プロンプト (改訂版)
【入力テキストの指示】
> 以下の #INPUT_TEXT# 部分に、分析・分解の対象となる階層的な構造を持つテキストデータ（シラバス、目次、議事録など）を挿入してください。
> 
【タスク定義と制約】
 * 要素の原子分解: 入力テキストに記載されているすべての概念、キーワード、複合語を、意味を保てる**最小の構成要素（原子要素）**まで徹底的に分解し、箇条書きの階層構造で表現してください。
   * 分解ルール: 句読点（、,）、結合文字（・-）、および括弧書き（()）内の要素は、すべて独立した最下層の原子要素として分離してください。
 * 番号付けの厳密な実行:
   * 親概念（最上位見出し、サブ見出し、グルーピング概念）には番号を振らないでください。
   * 分解によって生成された最下層の原子要素のみに対して、1から始まる連番を振り直し、カウントアップしていってください。この番号は最終出力まで一貫して維持してください。
 * 構造化とグルーピング:
   * 分解・番号付けされたキーワードを、**入力テキストの元の見出し（例: 1. 人工知能とは、4-2. ディープラーニングの概要）**の下に分類・グルーピングしてください。元の見出しはMarkdownの##で表現し、その下に階層構造でキーワードをネストさせてください。
   * 冗長性の解消と階層化: 分解とグルーピングの過程で、意味的に重複している概念や、階層化することで明確になる要素（例: 活性化関数の下にReLUをネストさせるなど）があれば、意味がより明確になるようにネスト（階層化）構造を積極的に適用し、リストの理解度と効率性を最大化してください。
