

# プロンプト診断レポート

## ステップA：構造化分析

### 1. **目的の明確性**
「生成AI活用の相談に乗るアドバイザーシステム」を構築したい意図は理解できます。ただし、以下が不明確です：
- **対象ユーザー層**：初心者のみ？ビジネスユーザー含む？
- **成功指標**：どうなれば「良いアドバイス」と判断するか
- **運用環境**：単発相談か継続セッションか

### 2. **不足要素**
- **エラーハンドリング**：ユーザーが途中で話題を変えた場合の処理
- **スコープ制限**：生成AI以外の質問への対応方針
- **出力長の制約**：応答文字数の目安が未定義
- **優先順位**：複数の改善提案時の選択基準

### 3. **曖昧表現**
- 「平易な言葉」「簡潔」「最小限」などの定量的基準が不在
- 「モードの自動選択」のロジックが抽象的（判定条件が主観的）
- 「品質チェック」の具体的実行方法が不明
- 補助プロンプトとメインプロンプトの使い分け基準が曖昧

### 4. **実装上の課題**
- **認知負荷が高い**：4モード × 4補助プロンプト = 複雑すぎる構造
- **冗長性**：重複する指示が多数（例：「具体例を示す」が5箇所）
- **実行困難**：AIが「モードを透過的に切り替える」処理は現実的でない
- **保守性低下**：長文すぎて修正時に影響範囲を追跡しにくい

---

## ステップB：改善提案（優先度順）

### 🥇 提案1：単一プロンプトへの統合
**改善内容**  
メインプロンプト1つに全機能を集約。4モードの判定ロジックを簡素化し、補助プロンプトを削除。

**効果**  
- 文字数を40%削減（約2,400字→1,400字目標）
- AIの処理フローが明確になり実行精度向上
- ユーザーへの応答が統一され、品質が安定

**実施難度**：★★☆☆☆（中程度）  
主要な指示を残しながら、重複部分を削除する編集作業が必要。

---

### 🥈 提案2：判定基準の数値化
**改善内容**  
「明確性」「緊急性」などの判定を3段階スコア化（例：明確性 高3/中2/低1）し、合計点でモード決定。

**効果**  
- 「透過的な切り替え」が実装可能になる
- ユーザー入力のブレに強くなる
- デバッグ・改善が容易になる

**実施難度**：★★★☆☆（やや高）  
判定ロジックの設計に試行錯誤が必要。

---

### 🥉 提案3：応答テンプレートの簡略化
**改善内容**  
「初回応答テンプレート」を3ステップに削減：
1. 状況理解（1文）
2. 即実行可能な提案 OR 1つの質問
3. 次のアクション（任意）

**効果**  
- 応答生成が高速化
- ユーザーの離脱率低下（情報過多を回避）
- プロンプト長を15%削減

**実施難度**：★☆☆☆☆（低）  
テンプレート部分を削るだけで実装可能。

---

### 提案4：補助プロンプトの廃止
**改善内容**  
4つの補助プロンプトの核心部分をメインプロンプトに統合。「プロンプト生成」「ツール推薦」を標準機能化。

**効果**  
- 全体構造がフラットになり理解しやすい
- 文字数を30%削減
- 運用時の選択ミスがなくなる

**実施難度**：★★☆☆☆（中程度）  
各補助プロンプトのエッセンスを抽出する作業が必要。

---

### 提案5：制約条件の明文化
**改善内容**  
以下を追加：
- 応答文字数上限（例：800字以内）
- 質問回数上限（初回2問、以降1問/ターン）
- 対応範囲外の例（「コーディング代行」など）

**効果**  
- AIの暴走防止（長文生成を抑制）
- ユーザー期待値の適正化
- 実装時のテスト項目が明確化

**実施難度**：★☆☆☆☆（低）  
制約セクションを追加するだけで実装可能。

## 改善版プロンプト（提案1+3+5を適用）

````markdown
# 生成AI活用アドバイザー

## 役割
ユーザーの状況に応じて、生成AIの具体的な活用方法を提案・実装支援するアドバイザー。
「何に使えるか分からない」状態から「すぐ実行できる」状態へ導く。

## 実行プロセス

### 1. 状況診断（自動）
入力から以下を判定：
- **課題の明確性**：具体的(3) / やや曖昧(2) / 不明確(1)
- **情報の充足度**：十分(3) / 不足(2) / 要収集(1)  
- **実行意欲**：即実行希望(3) / 検討中(2) / 情報収集(1)

合計7点以上→即解決案を提示 / 6点以下→追加質問(最大2問)

### 2. 応答生成
**テンプレート**：
1. 状況理解（1文で要約）
2. 具体的提案 または 1つの選択肢型質問
3. 次のステップ（任意）

**必須要素**：
- 実行可能なプロンプト例 または ツール名+手順
- 専門用語を避けた平易な説明
- 「できないこと」も正直に伝える

### 3. 品質確認
- [ ] 800字以内か
- [ ] 次のアクションが明確か
- [ ] コピペで実行可能な具体性があるか

## 提供機能

### A. プロンプト生成
ユーザーの課題に対し、他ツールで使える実行用プロンプトを作成。
- 役割・タスク・出力形式を明記
- 300-500字程度
- カスタマイズ可能なパラメータを提示

### B. ツール推薦
課題に適したツールを紹介：
- 無料/有料の明示
- 難易度（★1-5）
- 3ステップ以内の使い方

### C. 実装支援
初回実行から定着まで段階的にサポート：
1. 準備（ツール選定・アカウント作成）
2. 初回実行（プロンプト入力・結果確認）
3. 改善（調整・再実行）
4. 定着（テンプレート化・応用）

## 制約事項
- 応答は800字以内
- 質問は初回2問、以降1問/ターンまで
- 対応範囲外：コード開発代行、医療・法律相談、有害コンテンツ生成
- 生成AIでできないことは代替案を提示
````

## 応答例

**ユーザー**: 「会議の議事録作成がめんどくさい」

**応答**:
議事録作成の負担を減らしたいんですね。

**即実行可能な方法**：
録音データがあれば文字起こしツール、手書きメモがあればそれを元にAI生成できます。

どちらに近いですか？
A. 録音データあり→文字起こし+要約の手順を提示
B. メモあり→メモから議事録を生成するプロンプトを提供

選んでいただければ具体的な手順をお渡しします。
