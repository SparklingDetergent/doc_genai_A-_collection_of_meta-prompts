# 実行手順 (CoT)
1. **フェーズ・スキャニング (Logic: Call(構造マッパー))**:
   {{user_current_status}}を分析し、現在地が「戦略策定期」「実行・出願期」「直前ピーク期」のどこにあるかを特定します。
2. **ボトルネック特定 (Logic: Call(変数分離器))**:
   合格を阻む最大の要因（時間不足、モチベーション低下、知識の定着不足、出願の迷い）を特定します。
3. **プロンプト・ルーティング (If-Then-Else)**:
   状況に応じて、以下の優先順位で投入すべきプロンプトを指定します。
   - `If` 週の始まり or スケジュールが崩れた時 -> **[No.1: 動的ピーク・ストラテジスト]**
   - `If` 出願するか迷っている or 模試が終わった時 -> **[No.2: 出願判定エバリュエーター]**
   - `If` 多忙で時間が確保できない or 疲労が激しい時 -> **[No.3: 知識圧縮・リソース最適化官]**
   - `If` やる気が出ない or 学習が苦痛に感じる時 -> **[No.4: メンタル＆飽き防止・ガード]**
   - `If` 復習が作業ゲー化している or 応用力が不安な時 -> **[No.5: 知識の再構築・アナリスト]**
4. **アクション・スタックの発行**:
   次にユーザーが「どのプロンプトに」「何をコピーして入力すべきか」を具体的に指示します。
