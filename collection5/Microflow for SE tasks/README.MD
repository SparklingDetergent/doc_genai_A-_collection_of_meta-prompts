# Enhanced Request: High-Resolution SE Task Micro-Flows

## 1. Background & Precision Goal
本リクエストの目的は、フルリモートSEの業務を「原子単位のアクション」まで分解し、それらを機能別に再構成することです。抽象的な表現を排除し、1行につき1つの具体的な動作（タスク）を記述することで、プロセスの均質化と自己チェックの精度を極限まで高めます。

## 2. High-Resolution Task Micro-Flows
以下の4つのカテゴリーにおいて、重複を恐れず、時系列かつ微細な粒度でタスクをリストアップしてください。

### A. 社内環境構築・手順書整備フロー
1. 顧客環境の構成定義書の読み込み
2. 必要なリソース（サーバー・DB等）の払い出し申請
3. OS・ミドルウェアのインストール作業
4. インストール手順のコマンド履歴の保存
5. 社内環境独自の接続設定（IP・ポート開放等）の実施
6. 環境構築手順書の初版作成
7. 手順書に基づく「クリーンインストール再試行」による正確性検証
8. ツール化可能な定型作業（ログ監視、データ投入等）の特定
9. シェルスクリプト等による簡易ツール作成
10. ツールの実行手順書とトラブルシューティングの明文化

### B. 不具合調査・課題管理・見積もりフロー
1. 不具合事象の報告内容の確認
2. 社内検証環境での事象再現試行
3. デバッグログの抽出と特定エラーの切り出し
4. ソースコード上の該当箇所の特定
5. 修正案の仮説構築と影響範囲の特定
6. 課題管理票への調査結果と暫定/恒久対策の起票
7. 改修に必要な工程（設計〜テスト）の細分化
8. 各工程の工数算出（バッファ含む）
9. 上司向けの見積根拠資料の作成
10. スケジュール（WBS）への改修タスクの反映

### C. 開発ライフサイクル（設計〜テスト・リリース）フロー
1. 改修要件に基づく基本設計書の修正
2. 内部インターフェースの詳細設計書更新
3. 設計内容に対するセルフチェック（整合性確認）
4. コーディング（実装）の実施
5. 単体テスト仕様書の作成（正常・異常・境界値）
6. 単体テストの実行とエビデンス取得
7. 結合テスト環境へのモジュール配置
8. 結合テスト仕様書の作成と実行
9. 総合テスト（シナリオテスト）の実施
10. リリース手順書の作成（バックアップ・切り戻し手順含む）
11. 運用マニュアルの差分更新

### D. 自己マネジメント・調整フロー
1. WBSのタスク最小単位（最大1日以内）への分割
2. 当日の作業計画（ToDo）の確定
3. 予期せぬ割り込みタスクの記録とWBS再調整
4. 進捗率の客観的算出と上司への定期報告
5. レビュー対象資料の先行送付とポイント解説の作成
6. 会議体（レビュー等）の招集通知とアジェンダ送付
7. 指摘事項の議事録作成とタスクへの再注入
8. 顧客向け説明資料（Q&A等）の準備
9. 週次での「手順書更新漏れ」のセルフチェック
10. 自身の稼働実績と見積もりの乖離分析

## 3. Structural Requirement for Architect
アーキテクトは、上記の「1行単位のタスク」を元に、以下の構造で出力を整理してください。
* **Task Sequence**: 各フローにおける実行順序の定義。
* **Input/Output**: そのタスクを開始するために必要な情報と、完了時に残る成果物（ドキュメント、コード、ログ）。
* **Quality Gateway**: そのタスクが「正しく終わった」と判断するためのセルフチェック基準。

## 4. Constraints
* **粒度の維持**: 「設計する」といった曖昧な表現ではなく、「設計書にXXの項目を追記する」といった動作レベルまで落とし込むこと。
* **自己完結**: 他者への依存を最小限にし、自分一人で完結できる手順として定義すること。
