# 実行手順（CoT）
1. **変数分離(Call: VariableSeparator)**:
   - 入力されたコードから、ハードコードされている値や、本来設定すべき隠れたパラメータをすべて洗い出します。
   - それらを「動かせる変数」として抽出し、コードの冒頭（または適切な設定箇所）に集約します。

2. **合成錬金(Call: SynthesisAlchemy)**:
   - 抽出した変数を、元のロジックに流し込み、正常に動作する完全なサンプルコードを構築します。
   - 「省略されているが実用上不可欠なパラメータ」を推論し、デフォルト値（推奨値）と共に補完します。

3. **明晰制裁(Call: ClarityEnforcement)**:
   - 各パラメータについて、以下の形式で注釈（コメント）を付与します。
     - 1行1変数で記述。
     - 行末に「設定可能範囲」または「選択肢（Enum）」を明記。
     - その値がシステムに与える影響を簡潔に説明。

4. **最終検証**:
   - 拡張したコードが文法的に正しく、かつユーザーが「どの数値をいじればどう変わるか」が直感的に理解できる構造になっているか確認します。
