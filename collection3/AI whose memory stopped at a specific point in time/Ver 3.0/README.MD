# 概要
2段階の処理。初めに入力形式を示して処理。次に、入力形式を真似して出力する指示。

# 原案
``````
## Ver 2.0 について
同時に次の会話へのコンテキスト情報を出力する点で、メインの出力の質が低下しているように感じられた。
だから 段階を分けてつるくした方がいいかもしれない

↓

## Ver 3.0 について
ちょっと分けてみた。

コンテキスト情報を テンプレート化し 入力形式としてのみ認識させておき、
1ターン目の出力の後で 同じテンプレートで逆に出力するように指示する 分け方をした。

メイン出力の自由度は増えたと思う
しかし手数が増えたのでシンプルさが失われた。

# メリット
少ないコンテキストで会話を継続させるという目的に近づいた。

各エージェントに これを踏襲 させることで、コンテキストウィンドウの肥大化による性能劣化を抑えつつ、連続的に記憶を保った作業ができると踏んだ。
``````

``````
## Step 3 — コンテキスト更新 [文脈管理 × 自己進化]
- 今回のやり取りを既存コンテキストへ差分マージする

## 制約
- コンテキスト情報は **次回の自分が読む** ことを前提に書く（他者説明不要）
- コンテキストは肥大化させず、蒸留・更新を繰り返す
``````
